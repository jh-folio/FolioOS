# Current Market Dashboard and Watchlist Widgets Design

## Goal

Add TradingView-powered current-market widgets in two places:

1. Replace the dashboard's fixed Market Tape strip with a configurable current-market widget board.
2. Redesign the Watchlist tab so the page shows compact watchlist entries, and clicking an entry opens a focused company detail modal with TradingView widgets and Folio OS-collected news.

The widgets are current-market UI only. They must not become briefing evidence, market-memory evidence, or saved briefing snapshots.

## Product Principles

- **Current vs canonical separation**: TradingView widgets show current market state and never mutate saved briefing JSON, visual sidecars, Personal Overlay, or market memory.
- **Zero-config first**: The dashboard has a useful default layout without API keys or setup.
- **User control**: Users can choose what they want to see: indices, FX, commodities, rates, and watchlist company charts.
- **Local evidence stays local**: Watchlist news comes from Folio OS indexed RSS/articles, not TradingView Top Stories.
- **Graceful external failure**: If TradingView scripts are blocked or unavailable, the app shows a clear fallback and keeps existing local dashboard/watchlist data usable.

## TradingView Widget Choices

TradingView official widget documentation lists widgets suitable for this feature:

- Dashboard board:
  - `Market Overview`: best fit for grouped indices, FX, commodities, and rates.
  - `Advanced Chart`: best fit for a large focused chart selected by the user.
  - `Ticker Tape`: optional narrow strip for users who like a constantly visible market ribbon.
  - Later optional catalog entries: `Forex Cross Rates`, `Stock Heatmap`, `Economic Calendar`.
- Watchlist company modal:
  - `Advanced Chart`: primary company chart.
  - `Symbol Info`: company quote and key info summary.
  - `Company Profile`: optional company description panel.
  - `Fundamental Data`: optional fundamentals panel when available.

Do not use TradingView `Top Stories` by default because the requested news surface is Folio OS-collected local news.

TradingView iframe widgets require explicit container height. Folio OS must set stable responsive heights for every widget container to avoid collapsed or cut-off embeds.

## Dashboard Experience

### Current State

The dashboard currently renders `review.marketTape.items` as a fixed `.rv-tape` grid. Backend data is generated by `features/investment_review/service.py::build_dashboard_tape()`. This remains useful as a fallback, but it should no longer be the primary current-market experience.

### Target State

The dashboard hero stays unchanged. Below the hero, replace the fixed Market Tape strip with a `Market Widget Board`:

- Default layout:
  - Large `Market Overview` widget with tabs/groups:
    - US Indices: `FOREXCOM:SPXUSD`, `NASDAQ:IXIC`, `DJ:DJI`, `TVC:RUT`
    - Korea/Asia: `KRX:KOSPI`, `KRX:KOSDAQ`, optional `KRX:KOSPI200`
    - FX/Rates: `FX_IDC:USDKRW`, `TVC:DXY`, `TVC:US10Y`
    - Commodities: `TVC:GOLD`, `NYMEX:CL1!`, optional `TVC:SILVER`
  - Focus `Advanced Chart` widget defaulting to S&P 500.
  - Optional compact `Ticker Tape`, default disabled.
- User controls:
  - Add widget from allowed catalog.
  - Remove widget.
  - Change widget size: `wide`, `medium`, `compact`.
  - Change symbol/preset.
  - Reorder widgets using up/down buttons first; drag-and-drop is not required for v1.
  - Reset to default.
- Fallback:
  - If TradingView cannot load, show the existing local Market Tape grid using `review.marketTape`.
  - Show an explanatory external-widget unavailable state in the widget container.

### Settings Storage

Create a small JSON settings file:

```text
data/market-widget-settings.json
```

Shape:

```json
{
  "version": 1,
  "dashboard": {
    "widgets": [
      {
        "id": "market-overview-default",
        "type": "market_overview",
        "title": "Global Markets",
        "size": "wide",
        "preset": "global_core",
        "theme": "auto"
      },
      {
        "id": "focus-spx",
        "type": "advanced_chart",
        "title": "S&P 500",
        "size": "wide",
        "symbol": "FOREXCOM:SPXUSD",
        "interval": "D",
        "theme": "auto"
      }
    ]
  }
}
```

Only allow known widget types and known config fields. Unknown types are dropped and a warning is returned.

## Watchlist Experience

### Current State

The Watchlist page currently shows watchlist cards with latest news inline and a combined news list below. This makes the tab dense and forces news onto the page before the user chooses a company.

### Target State

The Watchlist tab becomes a compact command surface:

- Keep the keyword/company add input and chips.
- Replace news-heavy cards with compact entries:
  - Company/keyword name
  - Ticker if resolved
  - Sector/tag chips
  - Related local news count
  - Latest local news date
  - Actions: open detail, company analysis
- Remove the page-level `#watchlistNews` feed from the default view.

Clicking a watchlist entry opens a modal/drawer. Desktop uses a centered modal with a max width around 1100px. Mobile uses a full-screen drawer.

### Watchlist Detail Modal

Modal sections:

1. Header:
   - Name, ticker, market, sector
   - Related news count and latest date
   - Actions: company analysis, add memo, close
2. TradingView widgets:
   - Left/top: `Advanced Chart`
   - Right/below: `Symbol Info`
   - Optional tabs: `Company Profile`, `Fundamental Data`
3. Folio OS News:
   - Local RSS/articles matching that watchlist item
   - Source, date, title link, tags
   - Empty state if no collected news exists

Do not show TradingView Top Stories in the v1 modal. If a future option adds it, it must be visually separated from Folio OS-collected news.

### Watchlist Detail API

Add:

```text
GET /api/watchlist/detail?item=<watchlist item>
```

Response:

```json
{
  "item": "NVIDIA",
  "company": {
    "name": "NVIDIA Corporation",
    "ticker": "NVDA",
    "market": "US",
    "sector": "Technology",
    "tradingViewSymbol": "NASDAQ:NVDA"
  },
  "tags": ["Semiconductors", "AI"],
  "news": [],
  "newsCount": 5,
  "latestDate": "2026-06-24",
  "warnings": []
}
```

Reuse `watchlist_overview()` filtering rules where possible. Add a focused helper that resolves a single item and returns its local news without loading or rendering every watchlist item's detail.

## Symbol Mapping

TradingView symbols are not always the same as yfinance symbols. Add a small frontend/backend mapping helper, not a broad provider adapter.

Rules:

- US common stocks: map `NVDA` to `NASDAQ:NVDA` by default unless exchange is known.
- US index defaults use explicit TradingView symbols from preset config.
- Korean six-digit codes:
  - KOSPI default: `KRX:<code>`
  - KOSDAQ can also use `KRX:<code>` unless later testing shows another exchange prefix is required.
- FX/rates/commodities use explicit preset symbols.

If a symbol cannot be mapped, show a local info fallback and keep the local news visible.

## Frontend Architecture

Create:

```text
public/tradingview-widgets.js
```

Responsibilities:

- Render only approved widget types.
- Inject TradingView widget scripts lazily when a widget becomes visible.
- Clear/re-render widgets when symbol/config changes.
- Provide failure fallback hooks.
- Keep widget construction isolated from `public/app.js`.

`public/app.js` should own application state and event handling, then call this renderer. It should not contain large TradingView embed script blobs inline.

## UI Design

Use the existing Folio OS dashboard style: quiet, dense, work-focused. Do not create a marketing-style hero or oversized cards.

Dashboard:

- Widget board appears directly where `.rv-tape` is currently rendered.
- Widget controls should be compact icon/text buttons.
- Cards use 8px radius or existing tokens.
- Widgets must have stable min heights:
  - wide chart: 480px desktop, 360px tablet, 320px mobile
  - overview: 420px desktop, 360px mobile
  - compact tape: 72px

Watchlist:

- Default page is a compact grid/list.
- News is hidden until a company is selected.
- Modal must trap focus, close on Escape, and restore focus to the clicked card.
- Mobile modal uses full viewport with internal scroll.

## Error Handling

- TradingView blocked/offline: show widget unavailable state and leave local Folio OS data visible.
- Settings file invalid: return default settings plus warning; do not crash dashboard.
- Unknown widget config: drop that widget and return warning.
- Watchlist item unresolved: show item name, local news, and no TradingView widgets.
- Local news unavailable: show empty state; chart/info widgets can still load.

## Testing

Backend:

- Market widget settings default, validation, unknown widget drop, save/load roundtrip.
- Watchlist detail returns one item, local news, and resolved company metadata.
- No route writes to briefing JSON or visual sidecars.

Frontend static/unit tests:

- Dashboard render uses widget board before fallback tape.
- Watchlist cards do not inline latest news.
- Clicking card opens modal root and requests `/api/watchlist/detail`.
- TradingView renderer rejects unknown widget types.
- Widget containers have explicit height classes.

Manual/browser QA:

- Dashboard widgets render on desktop and mobile.
- External script failure fallback shows local Market Tape.
- Watchlist modal opens/closes, traps focus, and shows local news.
- Korean ticker watchlist item maps to a usable TradingView symbol or shows graceful fallback.
